Grupos e Fóruns

Dúvidas? Não tem problemas isso é normal, por isso estamos aqui para te ajudar, entre em contato utilizando o Google Groups, lá poderemos dar atenção a sua dúvida sobre a utilização do CapiravaJS.

Lembre-se que as issues do Github são mais destinadas a resolver bugs, problemas ou features do Framework. Gostariamos que não enviassem dúvidas sobre como utilizar, de preferência a utilizar o Google Groups do CapivaraJS pois assim podemos separar melhor o que é uma dúvida de o que é um problema que precisamos resolver, e não se preocupe tentaremos resolver as suas dúvidas o mais rápido que nossa equipe consegue.

Issues no Github

Caso você encontre algum problema, ou bug dentro do capivara gostariamos muito de saber. Por isso criamos um pequeno tutorial de como nos mandar issues no Github.

Questões ou Problemas?

Abrir uma issue não é a melhor maneira de resolver problemas e questões sobre a utilização do CapivaraJS, explicaremos melhor no próximo tópico quando se deve abrir issues, pois isso evita misturar problemas e questões com Bugs dificultando o trabalho de determinar qual a velocidade em que as requisições devem ser atendidas. A maneira mais rápida e eficiente de resolver isso é utilizando o Google Groups, devido a velocidade em que a equipe ou a comunidade pode resolver.

Issue ou Bug?

Se você encontrou um bug no código fonte, você pode nos ajudar submetendo uma issue em nosso Repositório do GitHub. Melhor ainda se essa issue vier com um Pull Request para fixar o problema. A equipe empenha-se muito na resolução e fechamento de todas as issues que são abertas, infelizmente algumas podem demorar mais tempo que outras para serem fechadas, mas estaremos investindo muito esforço para que o CapivaraJS alcance sempre as expectatívas dos usuários.

Melhorias na Documentação

Se você tiver uma sugestão para a documentação, você pode abrir uma issue e descrever o problema ou melhorias que você tenha. Caso você esteja disposto a resolver o problema de documentação, seja ele pequeno como alguns erros ortográficos ou grandes inserções de texto.

É muito importante criar issues comentando qual é seu objetivo e a sua melhoria de documentação, evitando assim trabalho duplicado para outras pessoas que também possam ter encontrado o mesmo problema.

Submissão de Issues

Antes de qualquer submissão de issue, não custa nada passar lá na sessão de issues e ver se já não foi aberta ou solucionada. Se a sua issue for um bug que ainda não tenha sido reportado a nós, abra a issue imediatamente. Isso ajuda a concentrar o foco do time de desenvolvimento, pois não será necessário procurar problemas e sim apenas concerta-los, fazendo com que bugs sejam resolvidos com mais eficiência.

A nova issue contem vários campos que gostaríamos que fossem preenchidos para entendermos melhor o problema e categorizar por nível de urgência.

Mesmo que já falamos anteriormente reforçar nunca é demais, então aqui vão algumas dicas de como melhorar a documentação da issue, para que ela vá direto ao ponto.

  • Explicações - Explique qual o motivo disso ser um problema para você
  • A versão do CapivaraJS - Não se esqueça de dizer qual é a sua versão utilizada, pois muitas vezes uma migração para uma nova versão do sistema já pode resolver.
  • Reprodução do erro- Nos mostre um exemplo do problema, pode ser utilizando JSFiddle ou Plunker (ou qualquer outro editor online) para entendermos com mais facilidade
  • Issues Parecidas - Alguma issue reportada anteriormente é relacionada com a sua? Nos avise.
  • Sugestão para Solução- Se você conseguiu resolver o problema você mesmo, avise também para ajudar outras pessoas.

Caso você tenha a solução da issue que está aberta, você pode ir à próxima sessão para aprender a como enviar Pull Requests ao CapivaraJS clicando aqui.
