Crie componentes rapidamente e sem grandes esforços de uma maneira
que funcione com todos os frameworks JavaScript.

Introdução

Atualmente existem vários frameworks que possibilitam a criação de componentes, entretanto tais componentes acabam ficando isolados em determinada tecnologia, fazendo com que uma possível migração seja mais complexa.

O CapivaraJS é um framework híbrido e suas diretivas são totalmente personalizáveis, tornando assim os componentes independentes da tecnologia utilizada.

Instalação

Para instalar a versão mais recente utilizando a ferramenta de linha de comando npm, execute o seguinte:

$ npm install --save capivarajs

Importando no seu projeto

Após a instação, é necessário importar o CapivaraJS no seu projeto, algo similar ao exemplo abaixo: 

HTML<script src="node_modules/capivarajs/dist/capivara.min.js"></script>

Webpack

Se estiver usando Webpack, você pode importar o capivarajs como no exemplo abaixo: 

JSimport capivara from 'capivarajs';

Components

Criando Componentes

Vamos aprender agora como registrar um componente CapivaraJS.


Para registrar um componente, você pode usar capivara.component(tagName, options). Por exemplo: 
JScapivara.component('my-component', {
    template: '<div>Uhaau! Esse é meu primeiro componente.</div>'
})



Uma vez registrado, o componente já pode ser usado, para isso é necessário declara-lo com um marcador (#meuMarcador). Por exemplo: 
HTML<my-component #meuMarcador></my-component>

Quando um componente é declarado, ele ainda não é iniciado, é através do marcador que vamos iniciar o componente. Exemplo: 
JScapivara.componentBuilder('meuMarcador').build();

O resultado final seria:
Uhaau! Esse é meu primeiro componente.

Objeto de configuração

O objeto configuração é responsável pela criação do seu componente, é nele que será definido as variáveis que serão aceitas, o template e controlador do mesmo. Exemplo:

JSconst options = {
    template  : STRING,
    constants : ARRAY,
    functions : ARRAY,
    bindings  : ARRAY,
    controller: FUNCTION
} 

capivara.component('my-component', options)

O exemplo acima mostra o formato do objeto de configuração, mas para entende-lo, vamos analisar individualmente suas propriedades.

[template] É aqui que será definido todo HTML do componente.
[constants] Definição das constantes que o componente aceitará, constantes são parâmetros sem referência.
[functions] Funções de "callback" que o componente aceitará.
[bindings] Defini os parâmetros que o componente aceitará, se um parâmetro é definido como binding, toda a alteração feita pelo componente será refletido no contexto do parâmetro, em outras palavras, o binding mantém a referência.
[controller] Através dessa propriedade é possível receber o escopo do seu componente, fazendo do mesmo as funcionalidades do seu componente.

Constantes

As constantes são parâmetros que você informa para um componente, o valor do parâmetro é lido apenas uma vez, então toda alteração do valor não será refletida no componente.

Para entendermos como passar um parâmetro do tipo constante para um componente, vamos criar um exemplo onde passaremos uma constante com o nome de titulo.

Primeiro, temos que informar o componente que iremos passar uma constante com o nome titulo. 
JScapivara.component('my-component', {
    template: '<div></div>',
    constants: ['titulo']
})

A partir de agora, o componente já está preparado para receber sua constante, como pode ser visto abaixo, o código HTML não sofre alterações. 

HTML<my-component #meuMarcador></my-component>

Você pode passar o valor da sua constante na inicialização do componente.

JScapivara.componentBuilder('meuMarcador')
        .constants({
            titulo: 'Meu incrível titulo'
        })
        .build();

Pronto, sua constante foi passada para o componente, toda constante está disponível na propriedade $constants do seu escopo, por exemplo:

JScapivara.component('my-component', {
    template: '<div></div>',
    constants: ['titulo'],
    controller: function(scope){

        scope.$onInit = function(){
            console.log(scope.$constants.titulo);
        }

    }
})

Funções

As funções são parâmetros que você informa para um componente, são utilizadas quando o componente quer export ações de "callback".

Para entendermos como funciona as funções do componente, vamos criar um exemplo onde passaremos uma função chamada onClickButton, que será executada quando o usuário clicar no botão disponível do componente.

Primeiro, temos que informar o componente que iremos receber uma função com o nome onClickButton. 
JScapivara.component('my-component', {
        template: '',
        functions: ['onClickButton']
    })

Perceba que seu código HTML não sofre alterações 

HTML<my-component #meuMarcador></my-component>

O componente está preparado para receber nossa função, pode passa-la na inicialização do componente

JScapivara.componentBuilder('meuMarcador')
        .functions({
            onClickButton: function(){
                console.log('Uhaau! Funcionou!!!');
            }
        })
        .build();

Pronto, sua função foi passada para o componente, toda função está disponível na propriedade $functions do seu escopo, e basta apenas executa-la, por exemplo:

JScapivara.component('my-component', {
        template: '<button type="button" cp-click="$functions.onClickButton()">Clique-me</button>',
        functions: ['onClickButton'],
        controller: function(scope){
        }
    })

Perceba que no exemplo acima usamos a diretiva cp-click a documentação dela será vista depois.

Bindings

Os bindings são parâmetros que você informa para um componente, você deve utilizar binding quando quiser que o componente mantenha a referência do parâmetro, em outras palavras toda alteração de valor feita pelo componente será aplicado no seu parâmetro, assim como toda alteração feita no valor do parâmetro será refletido no componente.

Para entendermos como funciona os bindings do componente, vamos criar um exemplo onde iremos simular um objeto chamado pessoa e nosso componente irá alterar o nome dessa pessoa.

Primeiro, temos que informar o componente que iremos receber um parâmetro do tipo binding chamado nome. 

JScapivara.component('my-component', {
            template: '',
            bindings: ['nome']
        })

É sempre valido reforçar o fato de que seu código HTML não sofre alterações 

HTML<my-component #meuMarcador></my-component>

O componente está preparado para receber nosso parâmetro, precisamos informa-lo na inicialização do componente, é necessário informar seu contexto atual, para que o componente possa atualiza-lo quando uma alteração acorrer.

JSlet pessoa = { nome: 'Mateus' };

capivara.componentBuilder('meuMarcador')
        .context(pessoa)
        .bindings({
            nome: 'nome'
        })
        .build();

Perceba que quando declaramos nosso binding, colocamos o nome do atributo que está no contexto, fazendo assim que a refêrencia entre os escopos seja criada.

JScapivara.component('my-component', {
            template: '<input type="text" cp-model="$bindings.nome"/>',
            bindings: ['nome'],
            controller: function(scope){
            }
        })

Perceba que no exemplo acima usamos a diretiva cp-model a documentação dela será vista depois.

Interpolação

A interpolação é usada pelo CapivaraJS para fornecer vinculação de dados para os elementos HTML.


Para compreendermos a interpolação, vamos criar um exemplo onde será apresentado em um elemento h1 a soma de dois números do nosso escopo.

JScapivara.component('my-component', {
    template: `
        <h1>Soma: [[ numero1 + numero2 ]]</h1>
    `,
    controller: function(scope){
        scope.numero1 = 10;
        scope.numero2 = 15;
    }
});

O resultado final seria:

Soma: 25

Funções

Criamos algumas funções que podem facilitar no seu desenvolvimento, abaixo vamos entender cada uma delas.

capivara.isArray

Determina se uma referência é um array, informe o valor a ser verificado por parâmetro. Exemplo: 

JSlet pessoas = [{ nome: 'João' }];
let resultado = capivara.isArray(pessoas);

console.log(resultado); // O resultado deve ser TRUE.

capivara.isObject

Determina se uma referência é um objeto, informe o valor a ser verificado por parâmetro. Exemplo: 

JSlet pessoa = { nome: 'Maria' };
let resultado = capivara.isObject(pessoa);

console.log(resultado); // O resultado deve ser TRUE.

capivara.isDate

Determina se uma referência é uma data, informe o valor a ser verificado por parâmetro. Exemplo: 

JSlet pessoa = { 
    nome: 'Maria', 
    aniversario: new Date('1996-11-13T23:00:00Z') 
};

let resultado = capivara.isDate(pessoa.aniversario);
    
console.log(resultado); // O resultado deve ser TRUE.

capivara.isElement

Determina se uma referência é um NodeElement, informe o valor a ser verificado por parâmetro. Exemplo: 

JSlet resultado = capivara.isElement(document.body);
        
console.log(resultado); // O resultado deve ser TRUE.

capivara.isFunction

Determina se uma referência é uma função, informe o valor a ser verificado por parâmetro. Exemplo: 

JSlet minhaFuncao = function(){ 
    //code 
};

let resultado = capivara.isFunction(minhaFuncao);
            
console.log(resultado); // O resultado deve ser TRUE.

capivara.isNumber

Determina se uma referência é um número, informe o valor a ser verificado por parâmetro. Exemplo: 

JSlet notaDoCapivara = 10;

let resultado = capivara.isNumber(notaDoCapivara);

console.log(resultado); // O resultado deve ser TRUE.

capivara.isString

Determina se uma referência é uma string, informe o valor a ser verificado por parâmetro. Exemplo: 

JSlet nome = 'CapivaraJS';

let resultado = capivara.isString(nome);

console.log(resultado); // O resultado deve ser TRUE.

capivara.merge

Faça a mesclagem de objetos em um único objeto. informe os parâmetros que serão mesclados como no exemplo abaixo: 

JSlet obj1 = {
    nome: 'Maria'
};

let obj2 = {
    idade: 25
};
    
let resultado = capivara.merge(obj1, obj2);
    
console.log(resultado); // O resultado deve ser {nome: "Maria", idade: 25}.

capivara.copy

Você pode fazer a cópia de um objeto para que seja criado uma nova referência. Exemplo: 

JSlet obj = {
    nome: 'Maria'
};

let resultado = capivara.copy(obj);

console.log(resultado); 

// O resultado deve ser um objeto idêntico, entretanto com outra referência: {nome: 'Maria'}.

capivara.replaceAll

Você pode fazer o replace em uma String de todas as ocorrências encontradas. Exemplo: 

JSlet str = 'Eu amo capivara. Capivara é demais.';
let resultado = capivara.replaceAll(str, 'capivara', 'CapivaraJS');

console.log(resultado); 

// O resultado deve ser: Eu amo CapivaraJS. Capivara é demais.

Eventos

É possível escutar e emitir eventos, o CapivaraJS te entrega as funções $on e $emit.

capivara.$on

Para começar a escutar um evento, basta informar seu nome e uma função para ser executada quando seu evento for emitido. 

JScapivara.$on('meuEvento', function(){
    console.log('Uhaau! Meu evento foi executado');
});

capivara.$emit

A função $emit faz com que todos os ouvintes do seu evento seja executado. Exemplo: 

JScapivara.$emit('meuEvento');
Você pode enviar vários parâmetros no seu evento, para isso basta informa-los ao emitir um evento, logo após do seu nome. 
JSlet meuParametro = 'CapivaraJS'; 
capivara.$emit('meuEvento', meuParametro);

Diretivas

cp-click

Você pode executar uma função do seu escopo. Exemplo: 

JScapivara.component('my-component', {
    template: `        
        <button cp-click="minhaFuncao()">Clique-me</button>
    `,
    controller: function(scope){

        scope.minhaFuncao = function(){
            console.log('Minha função foi executada.');
        };

    }
});

cp-model

Essa diretiva possibilita a interação do seu escopo com as input, quando usamos cp-model toda modificação de uma input é aplicada no escopo. Exemplo: 

JScapivara.component('my-component', {
    template: `        
        <input cp-model="pessoa.nome"/>

        <button type="button" cp-click="test()">Clique-me</button>
    `,
    controller: function(scope){

        scope.pessoa = {
            nome: 'Mateus'
        };

        scope.test = function(){
            console.log(scope.pessoa.nome);
        };

    }
});

cp-repeat

O cpRepeat te possibilita duplicar elementos html de acordo com um array do seu escopo. Exemplo: 

JScapivara.component('my-component', {
    template: `        
        <ul>
            <li cp-repeat="pessoa in pessoas">
                [[pessoa.nome]]
            </li>
        </ul>
    `,
    controller: function(scope){

        scope.pessoas = [
            {
                nome: 'João'
            },
            {
                nome: 'Maria'
            }
        ]

    }
});

cp-show

O cpshow te possibilita mostrar ou ocultar elementos de acordo com uma condição. Exemplo: 

JScapivara.component('my-component', {
    template: `        
        <h1 cp-show="mostrarTexto">Esse é um texto de exemplo.</h1>

        <button cp-click="alterar()">Clique para Mostrar</button>
    `,
    controller: function(scope){

        scope.mostrarTexto = false;

        scope.alterar = function(){
            scope.mostrarTexto = true;
        }

    }
});
